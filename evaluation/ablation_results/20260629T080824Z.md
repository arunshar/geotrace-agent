# Cache Ablation

- Golden set: 3 queries (model: claude-sonnet-4-6)
- The exact/semantic response cache is measured on identical repeats.

| config | pass | mean tokens/query | mean cost/query (USD) | total cost (USD) | cache-hit stages |
|---|---|---:|---:|---:|---:|
| cache ON | cold | 2870 | 0.026273 | 0.078819 | 1 |
| cache ON | warm | 0 | 0.000000 | 0.000000 | 4 |
| cache OFF | cold | 2529 | 0.023355 | 0.046710 | 0 |
| cache OFF | warm | 2330 | 0.021633 | 0.064899 | 1 |

**Repeat-query saving (cache ON).** On an identical repeated query the cache cuts per-query LLM cost by 100% (cold $0.026273 -> warm $0.000000) and tokens by 100% (2870 -> 0).

**Control (cache OFF).** With the cache disabled the warm repeat costs the same as the cold run ($0.021633), confirming the saving comes from the cache.

**Scope.** The cache fires only on repeated/near-duplicate queries. On the distinct-query golden set there are no repeats within a cold pass, so the cache provides no saving there; its value in production scales with the operational repeat rate. The in-flight tool deduplicator is implemented but not yet wired into the run path, so it is not ablated.
