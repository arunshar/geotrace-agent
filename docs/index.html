Abstract

GeoTrace-Agent is a multi-agent framework that answers natural-language questions over heterogeneous trajectory data (vessel AIS feeds, road networks, weather, satellite imagery) by combining large-language-model planning with deterministic time geography. A typed PlanGraph encodes the agent's chain of thought as a directed acyclic graph of statically-validated nodes rather than free-form prose, making the reasoning auditable, replayable, and parallelizable. A central orchestrator runs the graph under a hard token, tool-call, and wallclock budget, computes geometric truth with a Hägerstrand space-time prism kernel, and dispatches specialized agents that embed my prior algorithms for trajectory-gap detection and rendezvous finding. A kinematic validator guarantees that no region returned to a user is physically infeasible. Crucially, the system ships with first-class evaluation and observability: a golden-set offline harness that gates CI, an online shadow monitor that detects plan drift, OpenTelemetry tracing with tail-based sampling, and a per-stage cost ledger.

Deterministic-first, then synthesize

Even strong LLM agents hallucinate spatial relationships, miscompute distances, or skip the physics check entirely. GeoTrace-Agent takes the opposite stance: every spatially-decidable sub-problem is computed deterministically by a numerical kernel before any LLM is asked to synthesize. The LLM is used only where it is genuinely needed, planning and narration, while geometry, feasibility, and the core detection algorithms are exact and unit-tested.

Auditable reasoning

A typed DAG, not prose. Every step has a schema, dependencies, and a token budget; the whole plan is replayable.

Physical correctness

A space-time prism is the ground truth for what a mover can reach. Infeasible answers are rejected, not explained away.

Measured efficiency

A token and tool-cost stack (adaptive prompt compression plus a hybrid exact + semantic cache), instrumented end to end by a per-stage cost ledger.

System architecture

Typed PlanGraph chain-of-thought

The planner does not emit free-form prose. It returns a JSON object validated against a strict schema: each node is one of prism.compute, gaps.detect, rendezvous.tgard, rendezvous.dc_tgard, validate.kinematic, retrieve.semantic, summarize, carrying its dependencies, inputs, expected tokens, and a confidence prior. The orchestrator topo-sorts the graph, runs independent layers in parallel, rejects cycles before any work runs, and refuses a plan whose expected tokens exceed the budget. Because the trajectory is a typed graph, "did the agent do the right thing" becomes a structural check rather than string matching.

Specialized agents over a deterministic kernel

The geometric truth lives in a Hägerstrand prism kernel (geo-ellipses, minimum orthogonal bounding rectangles, dynamic region merge). Specialized agents embed my prior research as deterministic tools: a GapDetector extending STAGD with dynamic region merge and an Abnormal Gap Measure, and a RendezvousFinder extending TGARD and the dual-convergence DC-TGARD variant with bi-directional pruning. The prism kernel is exposed as a Model Context Protocol server, and the agent itself speaks JSON-RPC 2.0 Agent-to-Agent with a published capability card.

Three-layer efficiency stack

Every LLM call passes a single TokenOptimizer choke-point: adaptive prompt compression, prefix-cache-aware assembly, structured-output enforcement, and a per-call token clamp. Tool spend is reduced by parallel-safe execution and a hybrid exact-plus-semantic cache; an in-flight tool-call deduplicator is implemented but not yet wired into the run path.

Agent evaluation & observability

The part most teams skip. GeoTrace-Agent treats evaluation, observability, and cost as first-class, on three surfaces.

Figure 2. Three evaluation surfaces. Offline scoring gates CI; an online monitor catches plan drift on live traffic; tracing and a cost ledger make every run inspectable.

The correctness anchor is deterministic, not an LLM judge

Because the domain has ground truth, the core evaluator is the space-time-prism feasibility oracle, not an LLM judge: a claimed region is valid only if it lies inside the physically reachable prism. That is reproducible, has no judge variance, and is cheap enough to run on every output as a guardrail. The honest open piece is judging the free-text summary, where a calibrated LLM evaluator would belong.

Why this matters here: the trajectory is a typed graph, so tool-trajectory evaluation checks which algorithm the planner chose (for example TGARD versus DC-TGARD) and in what dependency order, not just the final answer.

What it does, end to end

Ask a question like "Could VESSEL-1234 have rendezvoused with VESSEL-9876 between 06:00Z and 12:00Z near 56N 162W?" The agent:

  1. plans a typed graph (compute two prisms → intersect with DC-TGARD → validate kinematics → summarize);
  2. computes a Hägerstrand prism for each vessel from its anchors and maximum speed;
  3. intersects the prisms to get the candidate rendezvous region, tightening the time window from both ends;
  4. gates the region through the kinematic validator, rejecting anything physically impossible;
  5. returns the answer, the region polygon, a full trace id, and the cost.

The live demo shows exactly this: the typed PlanGraph, a map of the prisms and the rendezvous region, the validator verdict, and the token cost. It runs CPU-only and falls back to a deterministic plan when no LLM key is set, so the geometry is always inspectable.

Built on, and extending, my research

GeoTrace-Agent is the agentic capstone of my PhD work. It embeds, as deterministic agents, the algorithms from my dissertation and recent papers:

STAGD-DRM

Space-time-aware gap detection with dynamic region merge (ACM TIST 2024), the GapDetector agent.

TGARD / DC-TGARD

Tightened-bound rendezvous detection with dual-convergence pruning (ACM SIGSPATIAL 2022), the RendezvousFinder agent.

Pi-DPM

Physics-informed diffusion anomaly model (SIGSPATIAL 2025) scores the Abnormal Gap Measure's data term.

Pi-GRPO

The human-in-the-loop queue exports preference triples that feed DPO/GRPO post-training in the sibling Pi-GRPO project, closing the agent-to-RL loop.

In other words: the classical time-geography core is mine, the agentic orchestration, the protocol surfaces, and the evaluation and cost instrumentation are the new system that wraps it.

Honest scope

  • GeoTrace-Agent is a research-engineering blueprint with production-shaped architecture (FastAPI, Postgres/PostGIS, Redis, Chroma, an OpenTelemetry collector, Tempo, Langfuse, a Kubernetes path), demonstrated as a public repository and a live CPU demo, not a system running at production traffic.
  • The efficiency numbers are reproduced from a live golden-set run and a cache ablation (checked in under evaluation/); the golden set is small and the in-flight tool deduplicator is implemented but not yet wired into the run path.
  • The anomaly data term calls a Pi-DPM with a small default config; a production deployment should load a checkpoint trained on the live trajectory distribution.
